Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

User Data

Upload Data with CSV file.

Example request:
curl --request POST \
    "http://localhost/api/user-data/upload" \
    --header "Content-Type: multipart/form-data" \
    --header "Accept: application/json" \
    --form "file=@/var/www/html/storage/app/csv/example.csv" 
const url = new URL(
    "http://localhost/api/user-data/upload"
);

const headers = {
    "Content-Type": "multipart/form-data",
    "Accept": "application/json",
};

const body = new FormData();
body.append('file', document.querySelector('input[name="file"]').files[0]);

fetch(url, {
    method: "POST",
    headers,
    body,
}).then(response => response.json());

Example response (202):

Retrieve a list of user data.

Example request:
curl --request GET \
    --get "http://localhost/api/user-data?first_name=A&last_name=A&age=30&gender=male&mobile_number=123&email=test%40gmail.com&city=London&login=A&car_model=BMW&salary=1000&limit=10&offset=0&sort_by=age&order_by=asc" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/user-data"
);

const params = {
    "first_name": "A",
    "last_name": "A",
    "age": "30",
    "gender": "male",
    "mobile_number": "123",
    "email": "test@gmail.com",
    "city": "London",
    "login": "A",
    "car_model": "BMW",
    "salary": "1000",
    "limit": "10",
    "offset": "0",
    "sort_by": "age",
    "order_by": "asc",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

Request      

GET api/user-data

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Query Parameters

first_name   string  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must not be greater than 50 characters. Example: A

last_name   string  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must not be greater than 50 characters. Example: A

age   integer  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must be at least 18. Must not be greater than 80. Example: 30

gender   string  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Example: male

Must be one of:
  • male
  • female
mobile_number   string  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must not be greater than 25 characters. Example: 123

email   string  optional  

Must be a valid email address. This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must not be greater than 100 characters. Example: test@gmail.com

city   string  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must not be greater than 100 characters. Example: London

login   string  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must not be greater than 50 characters. Example: A

car_model   string  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Example: BMW

Must be one of:
  • BMW
  • Mercedes
  • Ford
  • Toyota
  • Honda
  • Fiat
  • Peugeot
  • Nissan
  • Volkswagen
salary   integer  optional  

This field is required when none of first_name, last_name, age, gender, mobile_number, email, city, login, car_model, and salary are present. Must be at least 1000. Must not be greater than 5000. Example: 1000

limit   integer  optional  

Must be at least 1. Must not be greater than 1000. Example: 10

offset   integer  optional  

Must be at least 0. Example: 0

sort_by   string  optional  

Example: age

Must be one of:
  • first_name
  • last_name
  • age
  • gender
  • mobile_number
  • email
  • city
  • login
  • car_model
  • salary
order_by   string  optional  

Example: asc

Must be one of:
  • asc
  • desc
